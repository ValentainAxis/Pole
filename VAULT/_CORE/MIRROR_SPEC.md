# MIRROR (Gateway) — спецификация встраивания ИИ в среду GENESIS

Цель: превратить «ритуал чата» в «протокол среды».
Один и тот же интерфейс работает с локальным и онлайн ИИ, сохраняя следы в Vault.

---

## 0) Ключевая идея
Человек общается с MIRROR-интерфейсом, а не с моделью напрямую.

MIRROR всегда делает 3 шага:
1) Поднимает контекст из Vault (START, последние TRACE/PAUSE, ограничения).
2) Оформляет запрос в контрактном формате (режим + роль + границы).
3) Сохраняет результат как след (TRACE/PAUSE) в Vault.

---

## 1) Роли моделей
Локальный ИИ (Ollama):
- санитар контекста (сжатие, нормализация, упаковка)
- быстрый помощник по коду
- генерация следов (PAUSE/TRACE)
- приватный слой (локальные данные остаются локально)

Онлайн ИИ:
- широкий синтез
- сложные архитектурные развороты
- генерация больших блоков, когда нужно

MIRROR выбирает backend:
- auto: локальный по умолчанию, онлайн по явному переключателю
- local: всегда локальный
- online: всегда онлайн

---

## 2) Режимы плотности (трение ↔ устойчивость)
LIGHT (короткие задачи):
- 1 абзац контекста
- 1 следующий шаг
- след одной строкой

WORK (обычная работа):
- подтягивается последний TRACE проекта
- фиксируется короткий PAUSE (done/open/next)

DEEP (длинные контуры):
- расширенный контракт контекста
- строгие границы формата
- след с метками, ссылками, артефактами

---

## 3) Форматы
### 3.1 MIRROR REQUEST (внутренний контракт)
Минимальные поля:
- project: имя/идентификатор (например SEED_GODOT_2026-01-14)
- mode: light|work|deep
- backend: auto|local|online
- task: текст задачи (1–10 строк)
- constraints: ограничения (опционально)
- context: подтянутые фрагменты (START + last trace snippet)

### 3.2 MIRROR TRACE (выходной след)
В Vault/_TRACES:
- timestamp
- project
- mode/backend
- what_done (1–3 строки)
- what_open (1–3 строки)
- next_step (1 строка)
- links (file://… по желанию)

---

## 4) Архитектурное правило «один компас»
Единый вход: START.md (сцена).
MIRROR живёт как модуль, открывается из START одной ссылкой.
CORE остаётся переносимым, компас остаётся единым.

---

## 5) Минимальная реализация (MVP)
1) Скрипт mirror.sh принимает:
   mirror.sh <project> <mode> <backend>
2) Берёт последний trace проекта из Vault/_TRACES (если есть)
3) Собирает промпт-контракт
4) Вызывает Ollama (backend local/auto)
5) Пишет результат в новый TRACE-файл и обновляет TRACE_LAST проекта
